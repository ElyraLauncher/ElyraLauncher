# Code of Conduct

## Standards

Participation in ElyraLauncher should stay technical, respectful, and focused on improving the project. Good collaboration includes clear reports, reproducible evidence, constructive review, and direct discussion of tradeoffs.

Unacceptable behavior includes harassment, personal attacks, repeated low-effort disruption, publishing private information, or pressuring maintainers to bypass ROM-first validation rules.

## Enforcement

Maintainers may edit, hide, or close issues and pull requests that violate these standards. Serious or repeated violations may result in restricted participation.

## Scope

This code applies to project issue trackers, pull requests, discussions, and other official project spaces.
